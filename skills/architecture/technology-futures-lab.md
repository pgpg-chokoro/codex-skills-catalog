# Technology Futures Lab

## Metadata

| Field | Value |
|---|---|
| Category | Architecture |
| Difficulty | Intermediate |

| Tags | architecture, future-scenario, decision-making, creative |

## Purpose

技術選択の分岐シナリオや未来像を考察し、設計判断の理解を深める。

## Use Cases

- 技術選定の振り返り
- アーキテクチャ議論
- 勉強会や技術ブログ
- 将来の進化可能性を考えたいとき

## Inputs

- 現在のアーキテクチャ
- 重要な技術選択
- 代替案
- プロジェクトの制約

## Workflow

1. 重要な設計判断を抽出する。
2. 別の選択肢を設定する。
3. その世界線での利点と欠点を推測する。
4. 数年後のシナリオを作る。
5. 現実の設計判断から学べることを整理する。

## Outputs

- `future-scenarios.md`
- `alternate-architecture.md`
- `tradeoff-analysis.md`

## Implementation Idea

Architecture Alternate History、Software Evolution Predictor、Alternate History Refactor を統合したSkillとして扱う。

## Example

「RESTではなくGraphQLを採用していたら」「コンテナを使えなかったら」などの分岐を考え、設計への影響を整理する。
