## Purpose

ソフトウェア開発に奇妙な制約を1つ与え、その条件下で設計を再考する。

## Use Cases

- 勉強会
- ハッカソン
- 技術面接練習
- 設計発想トレーニング

## Inputs

- 作りたいソフトウェア
- 与える制約
- 使用可能な技術
- 評価したい観点

## Workflow

1. 制約を明確にする。
2. 使えない手段を整理する。
3. 代替設計を複数考える。
4. 実装戦略を作る。
5. 制約から得た学びをまとめる。

## Outputs

- `challenge-brief.md`
- `alternative-design.md`
- `lessons-learned.md`

## Implementation Idea

「DB禁止」「関数最大5行」「ネットワーク禁止」などの制約で、普段の設計を見直す。
