## Purpose

異なるフレームワークや技術を意図的に組み合わせ、新しい設計やプロダクトの発想を作る。

## Use Cases

- ハッカソン
- 技術ブログ
- 新規プロダクトの発想
- 勉強会

## Inputs

- 技術A
- 技術B
- 解決したい課題
- 制約条件

## Workflow

1. 各技術の強みを整理する。
2. 共通点と相違点を分析する。
3. 融合したアーキテクチャを考える。
4. ユースケースを作る。
5. 実現上の課題を整理する。

## Outputs

- `fusion-concept.md`
- `architecture-sketch.md`
- `feasibility-analysis.md`

## Implementation Idea

一見関係の薄い技術を組み合わせ、実現可能性より先に発想の幅を広げる。

## Example

SQLiteとGitを組み合わせ、履歴をSQLで検索できる開発分析ツールを考える。
