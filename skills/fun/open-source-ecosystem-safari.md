## Purpose

OSSエコシステムを生態系として探索し、中心プロジェクト、周辺ツール、依存関係を楽しく理解する。

## Use Cases

- 新しい技術領域を調べるとき
- 勉強会でOSSを紹介するとき
- 技術スタックの周辺知識を得たいとき

## Inputs

- 対象プロジェクト
- 依存関係
- 周辺ライブラリ
- コミュニティ情報

## Workflow

1. 中心となるプロジェクトを特定する。
2. 依存関係と共生関係を整理する。
3. 周辺ツールを分類する。
4. 生態系マップを作る。
5. 次に探索すべき技術を提示する。

## Outputs

- `ecosystem-map.md`
- `species-catalog.md`
- `exploration-guide.md`

## Implementation Idea

フレームワーク、プラグイン、周辺ツールを生態系に見立てて整理する。

## Example

あるWebフレームワークを中心種として、バリデーション、認証、ORM、サーバ実装との関係を整理する。
