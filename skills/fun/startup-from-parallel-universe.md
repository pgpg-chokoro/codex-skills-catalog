## Purpose

既存ソフトウェアを題材に、別の世界ではどんな組織が作っていたかを想像する。

## Use Cases

- 技術ブログ
- 勉強会
- OSS紹介
- 発想トレーニング

## Inputs

- 対象ソフトウェア
- 主な機能
- 設計の特徴
- 架空の前提条件

## Workflow

1. ソフトウェアの特徴を抽出する。
2. 架空の利用環境を設定する。
3. 架空組織のプロフィールを作る。
4. 技術選択の理由を物語化する。
5. 現実の設計判断との違いを整理する。

## Outputs

- `company-profile.md`
- `alternate-history.md`
- `founder-letter.md`

## Implementation Idea

技術分析と創作を組み合わせ、設計思想を別角度から理解する。

## Example

完全オフライン環境で使うツールとして再解釈し、同期よりローカル保存を重視した理由を説明する。
