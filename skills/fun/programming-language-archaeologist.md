## Purpose

古いプログラミング言語や忘れられた設計思想を発掘し、現代技術との関係を整理する。

## Use Cases

- 技術史の勉強会
- 言語設計の理解
- 新しい技術の背景を知りたいとき

## Inputs

- 言語仕様
- 歴史的資料
- サンプルコード
- 現代の比較対象

## Workflow

1. 時代背景を整理する。
2. 言語の特徴を抽出する。
3. 現代言語への影響を調べる。
4. 消えた理由や残った思想を分析する。
5. 現代に活かせる教訓をまとめる。

## Outputs

- `language-profile.md`
- `influence-map.md`
- `modern-lessons.md`

## Implementation Idea

古い言語を単なる過去の技術ではなく、未活用のアイデア集として分析する。

## Example

Smalltalkのライブ環境やオブジェクト指向思想を、現代の開発体験やAI支援開発と比較する。
