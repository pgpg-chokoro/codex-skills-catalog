## Purpose

アルゴリズム問題を個別解法ではなく、再利用可能な解法パターンとして学ぶ。

## Use Cases

- LeetCodeやAtCoderの学習
- 技術面接対策
- 動的計画法、BFS、DFS、Sliding Windowなどの理解

## Inputs

- 問題文
- 制約条件
- サンプル入出力
- 学習者のレベル

## Workflow

1. 問題を要約する。
2. 制約から使えそうなパターンを列挙する。
3. 最適な解法パターンを選ぶ。
4. 汎用テンプレートに変換する。
5. 類題と練習課題を提示する。

## Outputs

- `pattern-analysis.md`
- `solution-template.md`
- `practice-roadmap.md`

## Implementation Idea

問題を解くだけでなく、「なぜこのパターンなのか」を説明し、次に似た問題で使える判断基準を作る。

## Example

部分配列の最大長問題を Sliding Window の変種として説明し、使える条件と使えない条件を整理する。
